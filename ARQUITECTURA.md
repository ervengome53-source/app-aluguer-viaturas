RF PRINCIPAIS (MVP)
Para o primeiro incremento (MVP), o sistema deve ter as seguintes funcionalidades essenciais:

RF01 – Registo e autenticação de utilizadores (cliente e gestor)

RF02 – Pesquisar viaturas disponíveis (por modelo, categoria, preço)

RF03 – Realizar reserva de viatura

RF04 – Gerir reservas (consultar, cancelar)

RF05 – Devolução de viatura (registo pelo gestor)


2. Padrão Arquitectónico Escolhido

Arquitectura MVC (Model-View-Controller)

Justificativa:

* Separa claramente a lógica de apresentação (View) - Controle (Controller) - dados (Model)

* Ideal para aplicações web, que é o caso do sistema

* Facilita manutenção e testes

* Permite que diferentes membros da equipa trabalhem em camadas distintas
